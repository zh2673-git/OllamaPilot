# 全局预设词典

## 目录说明

此目录存放全局预设词典，所有文档索引时都会加载这些词典进行实体匹配。

## 文件结构

```
config/dictionaries/
├── global.json      # 通用实体（人名、地点、组织等）
├── tcm.json         # 中医术语（病症、方剂、药物等）
├── medical.json     # 现代医学（疾病、药物、检查等）
├── legal.json       # 法律术语（法律、法规、司法等）
├── finance.json     # 金融经济（银行、证券、保险等）
├── technology.json  # 科技技术（编程、AI、网络等）
└── custom.json      # 用户自定义（自由添加）
```

## 如何添加新领域词典

### 方法1：新建 JSON 文件（推荐）

1. 复制 `custom.json` 模板
2. 重命名为你的领域（如 `education.json`）
3. 编辑内容，添加专业术语
4. 重启程序即可自动加载

### 方法2：编辑现有词典

直接编辑对应的 JSON 文件，添加新词条。

## 词典格式

```json
{
  "metadata": {
    "name": "词典名称",
    "version": "1.0",
    "domain": "领域标识",
    "description": "词典描述",
    "created_at": "创建日期",
    "updated_at": "更新日期",
    "note": "备注说明"
  },
  "entities": {
    "类型1": ["词条1", "词条2", "词条3"],
    "类型2": ["词条4", "词条5", "词条6"]
  }
}
```

## 实体类型建议

通用类型：
- 人名、组织、地点、时间、数值
- 产品、概念、事件

专业领域类型：
- **医学**：疾病、药物、症状、检查、科室
- **法律**：法律、法规、司法、诉讼、主体
- **金融**：机构、市场、产品、指标、业务
- **科技**：语言、框架、数据库、AI、网络
- **中医**：病症、方剂、药物、症状、穴位

## 使用技巧

1. **预置丰富词典**：索引前填充专业术语，大幅提升索引速度
2. **分类清晰**：按类型组织词条，便于管理和查找
3. **定期更新**：根据索引结果补充缺失的术语
4. **全局共享**：所有文档共享这些词典，一次维护全局受益

## 注意事项

- 全局词典是**只读**的，不会被 LLM 修改
- LLM 学习的新实体会保存到**文档私有词典**
- 建议定期备份自定义词典文件
- 词条越多，索引速度越快（减少 LLM 调用）
